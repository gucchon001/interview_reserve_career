# Webhookデバッグチェックリスト

## 問題: 予約を作成しても `doPost` 関数が呼ばれない

実行ログに `doPost` が表示されない場合、WebhookがGASに到達していません。

## 確認手順

### ステップ1: 最新のデプロイを確認

1. GASエディタ > **公開** > **デプロイを管理** を開く
2. **最新のデプロイ（バージョン71）** を確認
3. **種類** が **ウェブアプリ** になっているか確認
4. **WebアプリのURL** をコピー

### ステップ2: TimeRex側のWebhook URLを確認

1. TimeRex管理画面 > **チーム設定** > **Integrations** > **Webhook** を開く
2. 設定されているWebhook URLを確認
3. **ステップ1でコピーしたURLと完全に一致しているか確認**
   - デプロイIDが一致しているか
   - URLの形式が正しいか

### ステップ3: Webhook URLを更新（必要に応じて）

1. TimeRex管理画面で既存のWebhook設定を**削除**
2. **追加** をクリック
3. ステップ1でコピーした**最新のWebアプリのURL**を貼り付け
4. **保存** をクリック
5. **セキュリティトークン**が表示されたら、それをコピー

### ステップ4: セキュリティトークンの設定（オプション）

セキュリティトークンが表示された場合：

1. GASエディタ > **プロジェクトの設定** > **スクリプト プロパティ**
2. **行を追加** をクリック
3. **プロパティ**: `TIMEREX_WEBHOOK_TOKEN`
4. **値**: コピーしたセキュリティトークン
5. **保存** をクリック

**注意**: セキュリティトークンを設定した場合、TimeRexからのWebhookにトークンが含まれている必要があります。設定しない場合は、Webhookは受け付けられますが、セキュリティ上推奨されません。

### ステップ5: テスト実行

1. GASエディタで `getWebhookUrl()` を実行
2. 実行ログで以下を確認:
   - デプロイ一覧
   - セキュリティトークンの設定状況
   - URL形式

### ステップ6: 実際の予約でテスト

1. TimeRexで予約を作成
2. GASエディタ > **実行** > **実行ログ** を開く
3. 以下が表示されるか確認:
   - `[doPost] ===== WEBHOOK RECEIVED =====`
   - `[WebhookHandler] ===== handleEventConfirmed CALLED =====`
   - `[SpreadsheetService] ===== appendInterview CALLED =====`

### ステップ7: uidlogの確認

スプレッドシートに `uidlog` シートが作成され、Webhook受信の記録が残ります。

## よくある問題

### 問題1: デプロイIDが古い

**症状**: 実行ログに古いバージョン（例: バージョン54）が表示される

**解決方法**:
1. 新しいデプロイを作成
2. TimeRex側のWebhook URLを最新のデプロイIDに更新

### 問題2: Webhook URLが一致していない

**症状**: `doPost` が呼ばれない

**解決方法**:
1. GASエディタ > **公開** > **デプロイを管理** で最新のWebアプリURLを確認
2. TimeRex側のWebhook URLと完全に一致しているか確認
3. 一致していない場合は、TimeRex側のURLを更新

### 問題3: セキュリティトークンの検証でブロックされている

**症状**: `[doPost] ===== SECURITY TOKEN VALIDATION FAILED =====` が表示される

**解決方法**:
1. GASのスクリプトプロパティで `TIMEREX_WEBHOOK_TOKEN` を確認
2. TimeRex側のセキュリティトークンと一致しているか確認
3. 一致していない場合は、TimeRex側で新しいトークンを生成して設定

## デバッグ用関数

### `getWebhookUrl()`
Webhook URLとデプロイ情報を表示

### `runWebhookConfirmedTest()`
モックデータでWebhook処理をテスト

### `testWebhookReception()`
Webhook受信の確認手順を表示
