# 手動予約・手動操作の扱い（運用整理）

**出典:** 2026年2月24日 エージェント業務効率化mtg メモ・文字起こし  
**目的:** 手動予約・手動変更時のフロー、Lステップ連携、KPI・リマインダー方針を一覧で整理する。

---

## 1. 手動が関わる場面の整理

| 場面 | 説明 | システム上の扱い |
|------|------|------------------|
| **電話・口頭で日程が決まった場合** | 候補者と電話等で日時を決め、Lステップの予約URLは使わない | 手動予約として扱う（下記フロー） |
| **管理画面からの手動登録** | オペレーターが管理画面の「手動予約登録」で入力して登録 | `registerManualBooking` → Googleカレンダー＋interviews（source=Manual） |
| **予約の変更・キャンセルを口頭で行った場合** | 電話等で「キャンセルしたい」「日時を変えたい」と伝えられた場合 | LステップURLは使わず、カレンダー・ポーターズを手動で更新 |

---

## 2. 手動予約時の正しいフロー（電話・口頭で日程が決まった場合）

会議で確認された運用は以下のとおり。

1. **Lステップの予約URLは使わない**  
   口頭で決まった予約については、Lステップ経由のTimeRex URLは使わない。

2. **Googleカレンダーに直接予約を入れる**  
   担当者のGoogleカレンダーに、面談予定を直接登録する。  
   - **推奨:** 管理画面の「手動予約登録」から登録する（`registerManualBooking`）。  
     これにより Googleカレンダー作成 ＋ interviews シート記録（source=Manual）＋ Slack 通知まで一括で行える。

3. **ポーターズのフェーズは手動で変更する**  
   予約・キャンセル・日時変更いずれも、ポーターズ上のフェーズは手動で更新する。

4. **TimeRexの「画面上部の予約URL」を直接共有しない**  
   Lステップを経由しないTimeRexの予約URLをそのまま候補者に渡すと、**LINE連携・リマインダー・タグ連携が一切動かない**。  
   予約導線は **必ずLステップ経由**（ボタンタップ → 予約URL送付）で行う。

---

## 3. 手動予約とLステップ・リマインダー

### 3.1 現状の仕様

- **LINE UID**  
  手動予約では、候補者がLステップの予約URLを踏んでいないため、**LINE UIDを取得できない**。  
  そのため `interviews.line_uid` は空で保存され、LステップAPI連携（友だち情報更新・タグ・リマインダ操作）は**自動では実行されない**。

- **リマインダー**  
  - 自動: Lステップ経由で予約した場合のみ、GASがトリガーURLに `meeting_date` 等を送り、Lステップ側でリマインダが動く。  
  - 手動: UIDが取れないため、**Lステップ側でリマインダを動かすには、友だち情報に面談日時を手動で追加する**必要がある（Lステップ管理画面などで該当友だちの「面談日時」を設定）。

- **理想（会議での合意）**  
  手動で予約完了した場合も、**LINEに完了通知が飛び、リマインダーが動く**ことが望ましい。  
  → 実現には、手動登録時に「候補者に紐づくLINE UID」をシステム側で特定できる必要がある（現状は未対応）。

### 3.2 今後の確認・検討事項（会議メモより）

- **手動で予約変更した場合に、LINEに完了通知が飛ぶか**  
  カレンダー管理画面の機能を使って、手動登録・変更時にLINE通知が飛ぶか確認する（原口担当）。  
- **ポーターズAPI解放後**  
  申し込みボタンのポストバックでUIDを取得し、ポーターズに自動連携する機能を実装すれば、**申し込み時点でUIDを取得**でき、手動予約時もLステップ連携がしやすくなる（理想に近づく）。

## 4. 手動予約の頻度とKPI

- **頻度**  
  電話で面談日程を取り付けるケースでは、**ほとんどが手動予約**になるため、手動予約時の自動連携（通知・リマインダー）ニーズは高い。

- **KPIの数え方**  
  - 手動予約分のKPIは **ポーターズ上で数える**。  
  - システム（GAS・スプレッドシート）側で手動予約を数値化することは**想定していない**。  
  - 管理画面の統計は「今日の面談数」「今週の合計面談数」など、interviews の source=TimeRex/Manual を合わせて表示するが、手動のみのKPI集計はポーターズ側で行う。

---

## 5. 管理画面の「手動予約登録」の役割と制限

- **役割**  
  電話・メール等で決まった日程を、Googleカレンダーと interviews シートに一括登録する。  
  - Googleカレンダーに面談イベント作成（Google Meet追加を試行）  
  - interviews に `source=Manual` で記録  
  - Slack 通知（設定していれば）

- **制限**  
  - **LINE UID は保存されない**（`line_uid` は空）。  
  - そのため **Lステップの友だち情報更新・タグ・リマインダ操作は自動では実行されない**。  
  - リマインダを付けたい場合は、Lステップ側で該当友だちの友だち情報に面談日時を**手動で追加**する運用で対応する。

- **仕様詳細**  
  - 仕様書: [docs/spec.md](../spec.md) の「2.2 手動登録」「4.2 手動予約」「8.3 手動予約登録フォーム」  
  - 実装: `Code.gs` の `registerManualBooking`、管理画面の手動予約モーダル（Admin.html）

---

## 6. Lステップ経由で予約する重要性（注意喚起）

- **Lステップを経由しない**TimeRexの予約URL（画面上部などにあるURL）をそのまま候補者に渡すと:  
  - LINE連携ができない  
  - リマインダー機能が使えない  
  - タグ連携が使えない  

- そのため、**予約は必ずLステップ経由**（ボタンタップ → 予約URL送付 → 候補者がそのURLから予約）で行う運用を徹底する。
