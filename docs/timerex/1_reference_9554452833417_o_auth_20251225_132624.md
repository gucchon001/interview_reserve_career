OAuthによる承認

TimeRex OAuthサーバーは、次の機能を提供します。

OAuth認証フロー
ステップ1
サードパーティのアプリは、
GET
リクエストを行うことで、ユーザーをTimeRexの承認エンドポイントにリダイレクトします。
GET https://timerex.net/oauth/authorize
承認エンドポイントには、次のクエリパラメータが必要です
パラメータ
必須
説明
client_id
Yes
TimeRexによって提供されるクライアントID。
response_type
Yes
code
の固定文字列を指定。
redirect_uri
Yes
登録時に使用されるリダイレクトURIと同じである必要があります。
scope
Yes
必要なスコープ。（スペースで区切られています） 空の文字列に設定すると、すべてのスコープが含まれます。
state
No
CSRF攻撃をチェックするために使用されるランダムな値。
例:
https://timerex.net/oauth/authorize?client_id=93d983d6-1031-404a-ba3e-0f07a95bf73a&redirect_uri=http%3A%2F%2F127.0.0.1%3A8000%2Fcallback&response_type=code&scope=&state=J-WnviV01ccIEbi0rsEcrHVbjNoQ4W8F7sRM2onWX
リダイレクト後、ユーザーには認証画面が表示されます。
ユーザーがリクエストを承認すると、redirect_uriにコールバックが送信されます。
ステップ2
コールバックには、次のクエリパラメータが含まれます。
パラメータ
説明
code
認証コード
state
リダイレクトを開始したときに設定された状態。 状態が一致しない場合は、CSRF攻撃を防ぐために、リクエストを無視する必要があります。
例:
https://third-party-app.com/callback?code=cdb99a63305d3b7ac953237fd3a4e1c8ad25987ca10aa2ae18d05&state=nCCOKdTMuWC0Vink7nXhIbUzlBTrzjpWZZ1F3Eqe

ステップ4
アクセストークンを使用して、TimeRexAPIを呼び出すことができます。
現在のユーザーの情報を取得するには、ユーザー情報エンドポイントに
GET
リクエストを送信します。
GET https://timerex.net/api/beta/user/me
承認ヘッダーでアクセストークンを渡します。
“Authorization” : "Bearer {access_token}"

ステップ6
アクセストークンを取り消すには、TimeRexトークン取り消しエンドポイントに対して
POST
リクエストを行う必要があります。
POST https://timerex.net/api/beta/token/revoke
承認ヘッダーでアクセストークンを渡します。
“Authorization” : "Bearer {access_token}"
