# Webアプリデプロイ設定ガイド

## 実行ユーザーの選択について

GASのWebアプリをデプロイする際、「次のユーザーとして実行」の設定で2つの選択肢があります：

1. **ウェブアプリケーションにアクセスしているユーザー**
2. **自分**

## 各設定の違い

### 「ウェブアプリケーションにアクセスしているユーザー」を選択した場合

- **実行ユーザー**: Webアプリにアクセスしたユーザーの権限で実行
- **アクセスできるユーザー**: 「Googleアカウントを持つ全員」のみ選択可能
- **メリット**: ユーザーごとに異なる権限で実行される
- **デメリット**: 
  - スプレッドシートへの書き込み権限が必要な場合、各ユーザーに権限を付与する必要がある
  - Webhookの場合、TimeRexから送信されるため、特定のユーザーがアクセスしているわけではない

### 「自分」を選択した場合

- **実行ユーザー**: 常にデプロイしたユーザー（管理者）の権限で実行
- **アクセスできるユーザー**: 「全員」が選択可能
- **メリット**: 
  - スプレッドシートへの書き込み権限が確実に保証される
  - Webhookの場合、TimeRexから送信されるため、この設定が適切
  - ユーザー認証が不要（Webhookが確実に到達する）
- **デメリット**: 
  - すべての実行が管理者の権限で行われる（この用途では問題なし）

## 推奨設定

### Webhookエンドポイントの場合

**推奨設定:**
- **次のユーザーとして実行**: **自分**
- **アクセスできるユーザー**: **全員**

**理由:**
1. WebhookはTimeRexから送信されるため、特定のユーザーがアクセスしているわけではない
2. スプレッドシートへの書き込み権限が必要なため、管理者の権限で実行する必要がある
3. 「全員」がアクセス可能にすることで、TimeRexからのWebhookが確実に到達する

### 予約画面・管理画面の場合

**推奨設定:**
- **次のユーザーとして実行**: **自分**
- **アクセスできるユーザー**: **全員**

**理由:**
1. スプレッドシートへの書き込み権限が必要なため、管理者の権限で実行する必要がある
2. ユーザー認証が不要で、誰でもアクセス可能
3. セキュリティは、Webhookのセキュリティトークンや、URLパラメータの検証で確保

## セキュリティについて

「自分」で実行し、「全員」がアクセス可能にしても、以下のセキュリティ対策により安全です：

1. **Webhookのセキュリティトークン**: TimeRexからのWebhookには、セキュリティトークンが含まれます
2. **URLパラメータの検証**: 予約画面や管理画面では、URLパラメータを検証します
3. **スプレッドシートの権限**: スプレッドシート自体の共有設定で、書き込み権限を制限できます

## 注意事項

- 「自分」で実行する場合、すべての実行が管理者の権限で行われます
- スプレッドシートへの書き込み権限が必要な場合は、この設定が必須です
- Webhookの場合、「ウェブアプリケーションにアクセスしているユーザー」では動作しません（TimeRexから送信されるため）

## まとめ

**このシステムでは、「自分」で実行し、「全員」がアクセス可能にする設定を推奨します。**

この設定により：
- Webhookが確実に到達する
- スプレッドシートへの書き込みが確実に実行される
- ユーザー認証が不要で、誰でもアクセス可能

セキュリティは、Webhookのセキュリティトークンや、URLパラメータの検証で確保されます。
