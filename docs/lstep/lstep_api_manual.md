## API連携の始め方

**API連携機能はオプション機能です。**

API連携機能は、友だちが外部システム上で行った行動（商品購入や問い合わせなど）をトリガーとして、次のようなLステップのアクションを動かせる機能です。

- 外部システム側で取得した情報の登録
- 友だちへのリマインドメッセージ送信

**例：** 外部サービスでの予約完了をトリガーにLステップのアクションを発動

API連携の画面の使い方は「API連携機能の画面について」マニュアルを参照してください。

## まずはAPI

本マニュアルは **API連携（外部 → L-Step へリクエスト送る）** に絞っています。  
認証トークン・エンドポイント・トリガーURL の設定と、GAS からの簡単なテスト（`runLStepApiTriggerTest` など）までをここで行います。  
LINE公式アカウントの Webhook URL 設定（L-Step 受信用 URL の切り替えなど）は別手順です。

## 設定手順

### ①エンドポイントを作成する

1. 左メニューから **API連携** を開き、**【エンドポイント設定】** タブをクリックする。
2. **【+新しいエンドポイント】** ボタンをクリックしてエンドポイントを作成する。
3. エンドポイント名をクリックすると、エンドポイントごとの設定画面に移動する。
4. 設定画面に移動したら、次にパラメータを作成する。

---

### ②パラメータを作成する

1. **【パラメータ管理】** タブをクリックする。
2. **【+新しいパラメータ】** ボタンをクリックする。
3. パラメータ登録画面で各項目を設定する。

| 項目 | 説明 |
|------|------|
| **① JSONキー** | 外部システムからのリクエストでパラメータ名として設定する文字列。<br>例：`event_type`、`event_name`、`user_name`、`past_participation_count` など |
| **② データ型** | 文字列・数字・日時のいずれかを選択する。 |
| **③ ラベル** | 管理用のラベル。アクション設定で扱うときに表示される。<br>例：イベント形式、イベント名、申込者名、過去の累計参加回数 など |

必要なパラメータが作成できたら、パラメータ登録は完了です。

---

### ③アクションを設定する

1. **【連携アクション】** タブをクリックする。
2. **【+連携アクションを追加】** ボタンをクリックする。
3. 連携アクション新規作成画面が開く。

| 項目 | 説明 |
|------|------|
| **① 連携アクション** | 管理用のアクション名を入力する。 |
| **② ステータス** | **稼働中**：APIが呼び出されるとアクションが発動する。<br>**停止中**：APIが呼び出されてもアクションは発動しない。<br>保存後も、一覧画面でステータスを切り替えられる。 |
| **③ 友だちの絞り込み** | アクションの実行対象となる友だちを絞り込む。<br>詳細は「[詳細検索（友だち検索の方法）](詳細検索マニュアル)」を参照。 |
| **④ アクション設定** | APIリクエストが送られたときに実行するアクションを設定する。 |

各項目の入力・設定ができたら **【保存】** ボタンをクリックする。作成したアクションは連携アクション一覧に表示される。

## API連携の絞り込み条件とアクション実行について

アクション設定では、友だちの絞り込み条件やアクションの埋め込み情報などに、登録したパラメータを使用できます。

### 絞り込み条件

絞り込み条件で **「外部API連携」** を選択できる。エンドポイント名をクリックすると、パラメータ管理で作成したパラメータのラベルが選択肢に表示される。

外部API連携の情報で設定できる条件は、種別「標準」の友だち情報と同様です。

| 条件 | 説明 |
|------|------|
| 完全一致 | 入力した文字と友だち情報が完全に一致している友だちが表示される。 |
| 部分一致 | 入力した文字を友だち情報に含んでいる友だちが表示される。 |
| 登録あり | 選択した友だち情報に何らかの登録内容がある友だちが表示される。 |
| 登録なし | 選択した友だち情報に登録内容がない友だちが表示される。 |
| 完全一致除外 | 入力した文字と友だち情報が完全に一致する友だちが除外され、それ以外が表示される。 |
| 部分一致除外 | 入力した文字を友だち情報に含んでいる友だちが除外され、それ以外が表示される。 |
| 数値比較 | 入力した数値に対する友だち情報に登録された数値の大小で友だちを表示する。<br>例：友だち情報「スコア」が「50」「より大きい」友だちを絞り込む。 |

その他の絞り込み条件の詳細は「[詳細検索（友だち検索の方法）](詳細検索マニュアル)」を参照。

### アクション設定

テキストに埋め込む情報と、タグ・友だち情報の操作でパラメータを使用できる。

- **テキストの埋め込み情報**：取得した情報を埋め込める。
- **タグ操作**：取得した情報をタグとして追加する、または取得した情報をもとにタグをはずす。
- **友だち情報操作**：代入・加算・減算する値として取得した情報を使用できる。日付の情報を友だち情報「年月日」に保存できる。
- **リマインダ操作**：ゴール日に取得した日付情報を代入できる。

詳細は「[アクション設定ボタンでできること](アクション設定マニュアル)」を参照。

## ④アクセストークンを取得する

リクエストには認証のため **トークン** が必要です。

- API連携画面の **「認証」タブ** からトークンを確認・コピーできる。
- 下記⑤の手順でもトークンをコピーできる。

---

## ⑤外部システムからAPIリクエストを送信する

外部システム側で、受信用エンドポイントに送るリクエストを用意する。

1. **API連携** → **【エンドポイント設定】** タブ → リクエストを送りたいエンドポイント名をクリックする。
2. エンドポイント詳細画面で **【cURL実行サンプル】** ボタンをクリックする。
3. cURL実行サンプルが表示される。**【コピー】** ボタンで実行サンプルをコピーできる。

**注意：**

- 【コピー】ボタンからコピーした場合、マスクされたトークンが実際のトークンに置き換わる。
- 該当のエンドポイントで作成したパラメータのキーはサンプルに記載されている。値にはサンプルの値が表示される。
- リクエスト例の `uid` の部分は、Lステップの `friend_id` を使用することもできる。外部システム側で `friend_id` が管理できる場合に利用する。
- 個別開発が必要な場合は、担当者までお問い合わせください。

外部システム側でリクエストを送る設定を行えば、API連携機能の設定は完了です。作成したエンドポイントに外部システムからリクエストが送られると、設定したアクションが実行されます。

## 簡単なテスト手順（GASから実行）

①〜⑤の設定ができたら、GAS から次のどちらかで動作確認できます。

| テスト | 関数 | 内容 |
|--------|------|------|
| **最低限の疎通** | `runLStepApiConnectivityTestMinimal` | GET でサーバー到達のみ確認。エンドポイント不要。 |
| **トリガー送信テスト** | `runLStepApiTriggerTest` | ⑤のトリガーURLへ POST（uid 付き）。エンドポイントで設定したアクションが動くか確認。 |

**トリガー送信テストのやり方**

1. ⑤の手順で**【cURL実行サンプル】**を開き、表示された **URL** をコピーする（例: `https://api.lineml.jp/v1/api-codes/690/triggers/c4faddc7-b837-4637-b481-eaab5777af2a`）。
2. GAS エディタで `runLStepApiTriggerTest` を実行する。
   - **URL をプロパティに設定する場合**: スクリプトプロパティに `LSTEP_TRIGGER_URL` を追加し、上記URLを設定してから `runLStepApiTriggerTest()` を実行。
   - **URL を引数で渡す場合**: エディタで `runLStepApiTriggerTest('ここにURL', 'テストするUID')` を呼ぶ関数を一時的に作り、それを実行。
3. ログで HTTP 2xx になっていれば成功。L-Step 管理画面で、該当友だちのタグ・友だち情報など、設定したアクションが反映されているか確認する。

**「友だちが見つかりません」の 404 になる場合（UIDはいるはずのとき）**

- トリガーURL（api-codes/690 など）が紐づく **LINE公式アカウント** と、その UID の友だちが **同じアカウント** か確認する。別アカウントの友だちUIDでは見つかりません。
- L-Step の「友だちリスト」で、**そのアカウント** を開き、該当友だちの **UID** をコピーして再テストする（画面表示のUIDと完全一致させてください）。
- **テスト専用**: API は `friend_id` も受け付けるため、`runLStepApiTriggerTest(undefined, '204179348', true)` のように friend_id で送ってトリガーが動くか確認することはできる。**本番の予約連携では Webhook 転送で届くのは UID のみ**なので、友だち情報更新・タグ付けは **UID で送る必要があり、friend_id は使わない**。

## 補足：エンドポイントが必要な場合・不要な場合（本プロジェクトでの整理）

上記①〜⑤がL-Step公式のAPI連携手順です。以下は、面談予約システム連携における利用形態の整理です。

| 利用形態 | エンドポイント | 必要な設定 |
|----------|----------------|------------|
| **API連携（トリガー連携）** | **必要** | ①エンドポイント作成 → ②パラメータ → ③アクション設定 → ④認証トークン → ⑤外部からそのエンドポイントのURLへリクエスト送信。パラメータに応じてLステップ側のアクション（リマインド・タグ・友だち情報など）が実行される。 |
| **REST API（友だち情報更新など）** | **不要** | 「認証」タブのトークンのみ。`https://api.lineml.jp/v1` のパス（例: `/friend/update`）に Bearer トークンで直接リクエストする。本プロジェクトの疎通テスト（`runLStepApiConnectivityTestWithUid`）や予約確定時の友だち情報更新はこちら。 |

※ Lステップの提供形態によっては、友だち情報更新も「API連携のエンドポイント」経由のみの場合は、上記REST APIではなくエンドポイントのcURLサンプル（URL・トークン）を使用してください。

### リクエスト先URLについて

本プロジェクトでは **`https://api.lineml.jp/v1`** をベースURLにしています（`Config.LSTEP_API_BASE_URL`）。エンドポイントの【cURL実行サンプル】で表示されるURLは、トリガーごとの形式です。例: `https://api.lineml.jp/v1/api-codes/690/triggers/c4faddc7-b837-4637-b481-eaab5777af2a`

### DNS error が出た場合（参考）

疎通テストで `DNS error` となる場合、リクエスト先ドメインが名前解決できていません。ベースURLを `https://api.lineml.jp/v1` にしている場合は解消している想定です。まだエラーになる場合は、L-Stepサポートまたは【cURL実行サンプル】のURLを確認してください。
