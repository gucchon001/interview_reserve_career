# L-step API 設定ガイド（TimeRex 予約 → タグ付与）

TimeRex で予約が確定したときに、L-step の友だち情報・タグを更新するための **L-step 管理画面での設定手順** です。GAS 側ではすでにトリガーURL へ `uid`・`meeting_date`・`meeting_url`・`meeting_cancel_url`・`tag` を送る実装があります。ここでは **L-step 側の「エンドポイント」「パラメータ」「連携アクション」** をどう設定すればタグが付与されるかを説明します。

---

## 前提

- **利用する方式**: トリガーURL（REST の `POST /friend/update` は本契約では 404 のため使用しない）
- **GAS の役割**: 予約確定時に `LStepApiService.triggerFriendUpdate(lineUid, { meeting_date, meeting_url, meeting_cancel_url, tag })` でトリガーURL に POST する
- **L-step の役割**: そのエンドポイントで「パラメータ」を受け取り、「連携アクション」でタグ・友だち情報・リマインドを実行する

---

## 設定の流れ（3ステップ）

| 順番 | 内容 | 設定場所 |
|------|------|----------|
| ① | エンドポイントを作成する | API連携 → エンドポイント設定 |
| ② | パラメータを作成する（GAS が送るキー名と一致させる） | 同じエンドポイント内 → パラメータ管理 |
| ③ | 連携アクションを設定する（タグ付与・友だち情報・リマインド） | 同じエンドポイント内 → 連携アクション |

---

## ① エンドポイントを作成する

1. L-step 管理画面にログインする。
2. 左メニューから **「API連携」** を開き、 **「エンドポイント設定」** タブをクリックする。
3. **「+新しいエンドポイント」** をクリックする。
4. エンドポイント名を入力する（例: **面談予約連携**）。
5. 保存する。作成したエンドポイント名をクリックし、そのエンドポイントの詳細画面に移動する。

---

## ② パラメータを作成する

GAS は次の JSON キーでトリガーURL に POST します。**L-step の「パラメータ管理」には、このキー名と型をそのまま登録**してください。

| JSONキー（GAS が送る名前） | 推奨データ型 | ラベル（管理用） |
|----------------------------|-------------|------------------|
| `uid` | 文字列 | LINEユーザーID（※L-step がリクエスト識別に使用。登録不要な場合あり） |
| `meeting_date` | 日時 | 面談日時 |
| `meeting_url` | 文字列 | ミーティングURL |
| `meeting_cancel_url` | 文字列 | キャンセル用URL |
| `tag` | 文字列 | 付与するタグ名 |

- **送信形式**: 上記キーは **リクエストボディの直下** と、cURL サンプルに合わせて **`params` オブジェクトの中** の両方に含めて送信します（`params` はオブジェクト型。配列ではありません）。

**GAS が送るリクエストボディの形式（参考）**

- **ルート直下**: `uid`, `meeting_date`, `meetingDate`, `面談日時`, `meeting_url`, `meetingUrl`, `ミーティングURL`, `meeting_cancel_url`, `meetingCancelUrl`, `キャンセル用URL`, `tag`, `付与するタグ名`（snake_case と camelCase の両方を含む。L-step の画面表示に合わせて camelCase も送信）
- **`params`**: 上記と同じキー・値（snake_case とラベル名）を持つ **オブジェクト**（配列 `[]` ではない）

- **params が空であることは不具合です。** GAS では、`meeting_date` 等の data があるのに `params` が空の場合は送信前にエラーにし、送信しません。本番では実行ログの「送信 params キー数」「送信 params 内容」で、L-step に送っている `params` オブジェクトの中身を確認できます。L-step の画面で `params: []` と見える場合は、GAS ログで実際の送信内容を確認し、L-step 側の受信仕様に問い合わせることを推奨します。

**単体では params が入るのに本番で入らない場合の原因候補**

| 原因 | 確認方法 |
|------|----------|
| **本番が古いデプロイのまま** | GAS の「バージョン」で本番にデプロイされているスクリプトが、`buildTriggerPayload` で `params` オブジェクトを組み立てる最新版か確認する。 |
| **本番で渡している data が空** | 実行ログで `[WebhookHandler] L-step に渡す data キー数: 4` が出ているか確認。4 なら meeting_date / meeting_url / meeting_cancel_url / tag を渡している。 |
| **送信はしているが L-step が表示していない** | ログで `[LStepApiService] 送信 params キー数: 8` および `送信 params 内容: {"meeting_date":"...", ...}` を確認。ここに中身が出ているのに L-step 画面で `params: []` なら、L-step 側の受信・表示仕様の可能性が高い。 |

**手順**

1. 対象エンドポイントの **「パラメータ管理」** タブをクリックする。
2. **「+新しいパラメータ」** で、上表のとおり **① JSONキー**・**② データ型**・**③ ラベル** を設定する。
3. `uid` は cURL サンプルに元から含まれることが多いため、既にある場合は重複登録しない。
4. 必要なパラメータをすべて登録したら保存する。

---

## ③ 連携アクションを設定する（タグ付与・友だち情報・リマインド）

ここで「トリガーが呼ばれたときに何をするか」を設定します。

1. 同じエンドポイントの **「連携アクション」** タブをクリックする。
2. **「+連携アクションを追加」** をクリックする。
3. 以下を設定する。

| 項目 | 設定例 |
|------|--------|
| **① 連携アクション** | 例: 面談予約確定時のタグ付与 |
| **② ステータス** | **稼働中** |
| **③ 友だちの絞り込み** | 「外部API連携」で、当該エンドポイントの **uid**（または送信元の識別子）が「登録あり」など、必要に応じて設定。通常は「絞り込みなし」で全員に実行してもよい。 |
| **④ アクション設定** | 次項のとおり。 |

### アクション設定で行うこと

- **タグ操作（推奨: 固定タグ）**  
  - **「タグを追加」** を選び、付与するタグで一覧から **「面談予約済み」** を選択する。  
  - トリガーが発火するたびに、その友だちに「面談予約済み」が確実に付与されます（実運用で確認済み）。
- **タグ操作（代替: 連携データから）**  
  - 「連携データからタグを追加」で **パラメータ `tag`**（または「付与するタグ名」）を選ぶ方法もありますが、環境によってはタグが付かない場合があります。その場合は上記の「タグを追加」に切り替えてください。
- **友だち情報操作**（任意）  
  - 面談日時・URL を友だち情報欄に残したい場合:  
    - 友だち情報「面談日時」に **パラメータ `meeting_date`** を代入  
    - 友だち情報「ミーティングURL」に **パラメータ `meeting_url`** を代入  
    - 友だち情報「キャンセルURL」に **パラメータ `meeting_cancel_url`** を代入  
  - 事前に「友だち情報欄管理」で「面談日時」「ミーティングURL」などの項目を作成しておく。
- **リマインド操作**（任意）  
  - 面談前にリマインドを送る場合: リマインドの「ゴール日」に **パラメータ `meeting_date`** を設定する。

設定後 **「保存」** をクリックする。

### タグが付かない場合の切り替え手順

「連携データからタグを追加」にしているがタグが付かない場合は、**「タグを追加」（固定タグ）** に切り替えてください。

1. 連携アクションの **「アクション設定」** を開く。
2. **タグ操作** で **「タグを追加」** を選ぶ。
3. **付与するタグ** で一覧から **「面談予約済み」** を選択する。
4. **「この条件で決定する」** で保存する。

この設定で、本番予約フロー（TimeRex 確定 → Webhook → トリガーURL）でもタグ付与が動作することが確認されています。

---

## ④ GAS に設定する値（トリガーURL とトークン）

L-step 側の設定が終わったら、GAS からそのエンドポイントを呼び出すために以下を設定します。

1. **トリガーURL の取得**
   - **API連携** → **エンドポイント設定** → 作成したエンドポイント名をクリック。
   - **「cURL実行サンプル」** ボタンをクリックし、表示された **URL** をコピーする（例: `https://api.lineml.jp/v1/api-codes/690/triggers/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`）。
2. **GAS のスクリプトプロパティ**
   - **`LSTEP_TRIGGER_URL`**: 上記でコピーした**予約用**トリガーURL をそのまま貼り付ける。
   - **`LSTEP_CANCEL_TRIGGER_URL`**（任意）: キャンセル用エンドポイントを別に作成した場合、その cURL サンプルの URL を設定。未設定時は予約用 URL が使われ、キャンセル時にも予約時メッセージが送られる可能性あり。詳細は [LSTEP_SPEC_GUIDE.md](LSTEP_SPEC_GUIDE.md) を参照。
   - **`LSTEP_API_TOKEN`**: 「認証」タブまたは cURL サンプルで表示されるトークンを設定（既に設定済みの場合はそのままでよい）。

これで、TimeRex で予約が確定すると GAS がトリガーURL に POST し、L-step 側の連携アクションが実行されて **タグが付与**されます。

---

## 動作確認

1. **GAS のみの疎通確認**
   - エディタで `runLStepApiTriggerTestForSpecifiedUid` を実行する（または `runLStepApiTriggerTest(トリガーURL, 'テスト用UID')`）。
   - ログで HTTP 2xx になっていれば、L-step までリクエストは届いている。
   - **注意**: このテストは **uid のみ** 送るため、L-step の「連携データからタグを追加」ではタグは付きません。タグ付与まで確認するには次項の「タグ付きテスト」を使います。
2. **タグ付きテスト（タグが付くか確認）**
   - エディタで **`runLStepApiTriggerTestWithTag`** を実行する。引数なしならサンプルUID、または `runLStepApiTriggerTestWithTag('あなたのテスト用UID')` で指定。
   - 本番と同じく `uid` と `tag`（例: 面談予約済み）を送るため、L-step で「連携データからタグを追加」が設定されていれば、該当友だちにタグが付きます。
3. **タグ・友だち情報の確認**
   - L-step の「友だちリスト」で、テストに使った UID の友だちを開く。
   - 連携アクションで設定した「タグ」や「友だち情報」が更新されているか確認する。
   - タグ一覧に「面談予約済み」が出ていない場合は、L-step の「タグ」画面で「新規タグ名」に「面談予約済み」を入力して「追加」し、必要ならタグフォルダを指定してから再度テストする。
4. **本番フロー**
   - LINE から予約リンク（`?from=line`）で予約画面を開き、TimeRex で予約を確定する。
   - 同じ友だちのタグ・友だち情報が更新されていれば、E2E で問題なく連携できています。

## 補足

- **テキスト送信のプレースホルダ**: GAS は `meeting_date` / `meeting_url` / `meeting_cancel_url` に加え、L-step のラベル名（面談日時・ミーティングURL・キャンセル用URL）をキーにした値も送っています。メッセージで「面談日時」「ミーティングURL」「キャンセル用URL」の連携データを埋め込んでいる場合は、これで正しく表示されます。
- **タグ名の変更**: GAS の `Config.LSTEP_TAG_NAMES.BOOKING_CONFIRMED`（既定: `面談予約済み`）を変更すると、送られる `tag` の値が変わります。L-step 側で「タグ操作」に「取得した情報（パラメータ `tag` または 付与するタグ名）」を指定していれば、同じタグ名が付与されます。
- **タグが付かない場合の確認**: ①連携アクションのステータスが「稼働中」か。②「友だちの絞り込み」で対象が除外されていないか（テスト時は「絞り込みなし」を推奨）。③L-step の「タグ」で「面談予約済み」を事前に作成し、タグフォルダ「API連携」を指定しているか。④GAS は `tag` と `付与するタグ名` の両方のキーで同じ値を送るため、L-step がどちらのキーを参照していても値が渡る。
- **友だち情報・タグを更新したくない場合**: `Config.LSTEP_UID_ONLY = true` にすると、GAS は UID 取得のみ行い、トリガーURL は呼びません。
- **予約キャンセルとリマインド解除**: 予約キャンセルは TimeRex の機能で実行される。キャンセル時に GAS が **キャンセル用トリガーURL**（`LSTEP_CANCEL_TRIGGER_URL`。未設定時は予約用 URL）に `meeting_date: null` 等を送ることで L-step の友だち情報がクリアされ、リマインド条件から外れて配信が停止する。**キャンセル時に予約時メッセージを送らない**ためには、L-step にキャンセル用エンドポイントを1本用意し、連携アクションは「友だち情報の更新のみ」とすること。詳細は [LSTEP_SPEC_GUIDE.md](LSTEP_SPEC_GUIDE.md) を参照。責任分界とフローは [TimeRex・GAS・L-step 責任分界とリマインド連携設計](../design/TIMEREX_GAS_LSTEP_DESIGN.md) を参照。
- **詳細な API 手順**: [L-step API連携マニュアル](lstep_api_manual.md) の ①〜⑤ も参照してください。
- **疎通・契約の注意**: [LSTEP_CONNECTIVITY_SUMMARY.md](LSTEP_CONNECTIVITY_SUMMARY.md) に、REST 404・トリガーURL 利用の説明があります。
